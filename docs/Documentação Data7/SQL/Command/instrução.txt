Esse é o autocomplete completo da classe SQL.Command, com base nele, atualize a definição completa da classe.
Note que tem a coluna Suportado Sim/Não, ela indica se conseguimos usar esse membro da classe no projeto, 
se tiver marcado como não, adicione a definição na classe, mas deixe ela marcada para gerar um erro no linter
caso seja usada, pois não teremos suporte à elas ao compilar o projeto.

Cruze os dados com as classes que o SQL.Command herda, para não adicionar props que já existem nas classes base

Além dessa definição do autocomplete, analise a pasta Documentação Data7/SQL/Command, nela vão ter os arquivos
XML das documentações das classes nativas base TFDQuery e da SQL.Command,  portanto SQL.Command terá tudo que existe na documentação
das classes bases e que exista nessa definição de autocomoplete abaixo:
