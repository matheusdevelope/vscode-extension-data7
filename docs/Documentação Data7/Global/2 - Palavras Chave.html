2 - Sintaxe Básica

Nº do Artigo: 199 | Avaliação: Não avaliado | Última atualização: Fri, May 10, 2019 at 10:30 AM

O Developer Studio utiliza como sintaxe para programação, a linguagem VB.Net, que é uma linguagem de programação orientada a objetos. Na metodologia de Programação Orientada a Objetos, um programa consiste em vários objetos que interagem entre si por meio de ações. As ações que um objeto pode executar são chamadas de métodos. Dizem que objetos do mesmo tipo têm o mesmo tipo ou, mais freqüentemente, são da mesma classe.

Quando consideramos um projeto no Developer Studio, ele pode ser definido como uma coleção de objetos que se comunicam através da invocação dos métodos uns dos outros. Vamos agora examinar brevemente o que significa classe, objeto, métodos e variáveis ​​de instância.

  • Objeto - Objetos possuem estados e comportamentos. Exemplo: Um cão tem estados - cor, nome, raça, bem como comportamentos - abanar, latir, comer, etc. Um objeto é uma instância de uma classe.

  • Classe - Uma classe pode ser definida como um template / blueprint que descreve os comportamentos / estados que os objetos de seu tipo suportam.

  • Métodos - Um método é basicamente um comportamento. Uma classe pode conter vários métodos. É nos métodos onde as lógicas são escritas, os dados são manipulados e todas as ações são executadas.

  • Variáveis ​​de instância - cada objeto tem seu conjunto exclusivo de variáveis ​​de instância. O estado de um objeto é criado pelos valores atribuídos a essas variáveis ​​de instância.

Uma classe de retângulo no Developer Studio

Por exemplo, vamos considerar um objeto Retangulo. Esse objeto tem atributos como comprimento e largura, e dependendo do design, pode ser necessário aceitar os valores desses atributos, calculando a área e exibindo os detalhes.

Vamos olhar para uma implementação de uma classe Retangulo e discutir a sintaxe básica do Developer Studio com base em nossas observações.

Public Class Retangulo
   Private tamanho As Double
   Private largura As Double
 
   'Métodos Publicos
   Public Sub Medidas()
      tamanho = 4.5
      largura = 3.5
   End Sub
 
   Public Function GetArea() As Double
      GetArea = tamanho * largura
   End Function
 
   Public Sub Mostrar()
      Print "Tamanho: "& tamanho.ToString()
      Print "Largura: " & largura.ToString()
      Print "Area: " & GetArea().ToString()
   End Sub
 
End Class
 
Dim ret As New Retangulo()
ret.Medidas()
ret.Mostrar()

Uma classe pode ter membros que podem ser acessados ​​de fora da classe, se assim for especificado. Os membros de dados são chamados de campos e os membros de procedimento são chamados de métodos.

Métodos compartilhados ou métodos estáticos podem ser chamados sem criar um objeto da classe. Métodos de instância são invocados por meio de um objeto da classe.

Identificadores

Um identificador é um nome usado para identificar uma classe, variável, função ou qualquer outro item definido pelo usuário. As regras básicas para nomear classes no VB.net são as seguintes:

  • Um nome deve começar com uma letra que pode ser seguida por uma seqüência de letras, dígitos (0 a 9) ou sublinhado. O primeiro caractere em um identificador não pode ser um dígito.

  • Não deve conter nenhum espaço ou símbolo incorporado? - +! @ #% ^ & * () [] {}. ; : "'/ e \. No entanto, um sublinhado (_) pode ser usado.

  • Não deve ser uma palavra reservada.
